# Fama 数据脱密模块设计方案

## 目标用户

法律共同体（律师、法官、法务、法学研究者），不懂技术。

## 设计原则

1. **零学习成本** - 打开即用，无需配置
2. **一键操作** - 上传文档 → 自动脱密 → 下载结果
3. **法律场景优先** - 预设法律文书常见敏感信息规则
4. **可视化反馈** - 清晰展示脱密前后对比

## 功能设计

### 核心流程

```
用户上传文档 → 自动检测敏感信息 → 显示预览 → 用户确认 → 生成脱密文档
```

### 技术实现

#### 模块结构

```
packages/masking/
├── src/
│   ├── engine.ts          # 脱密引擎核心
│   ├── detectors/         # 检测器
│   │   ├── id-card.ts     # 身份证检测
│   │   ├── phone.ts       # 手机号检测
│   │   ├── name.ts        # 姓名检测
│   │   ├── address.ts     # 地址检测
│   │   └── bank-card.ts   # 银行卡检测
│   ├── replacer.ts        # 替换策略
│   ├── entity-manager.ts  # 实体一致性管理
│   └── parsers/           # 文档解析
│       ├── docx.ts        # Word解析
│       ├── pdf.ts         # PDF解析
│       └── txt.ts         # 纯文本解析
├── ui/
│   ├── MaskingPanel.tsx   # 主面板组件
│   ├── PreviewView.tsx    # 预览组件
│   └── RuleSelector.tsx   # 规则选择器
└── rules/
    └── legal-default.json # 法律场景默认规则
```

#### 与 Fama 集成

1. **侧边栏入口** - 在 Fama 侧边栏添加"数据脱密"图标
2. **工具集成** - 作为内置工具，Agent 可以调用
3. **快捷命令** - `/mask` 命令快速启动

## 用户体验流程

### 场景1：律师脱密当事人信息

1. 律师打开 Fama
2. 点击侧边栏"数据脱密"图标
3. 将起诉状 Word 文档拖入上传区域
4. 系统自动检测：原告姓名、身份证号、联系方式等
5. 律师确认脱密范围（可取消勾选不需要脱密的项目）
6. 点击"下载脱密文档"
7. 获得脱密后的文档，可安全分享

### 场景2：法官批量脱密判决书

1. 法官选择多个判决书文件
2. 系统批量处理，显示进度
3. 生成脱密后的文档包
4. 一键下载

## 后续扩展

1. **自定义规则** - 用户可添加特定脱密规则
2. **模板保存** - 保存常用脱密配置
3. **批量处理** - 支持多文件同时脱密
4. **审计日志** - 记录脱密操作历史

---

## 技术选型

- **前端**：SolidJS + TailwindCSS（与 Fama 一致）
- **后端**：TypeScript（集成到 Fama 核心）
- **文档解析**：
  - Word: mammoth.js 或 docx
  - PDF: pdf-parse
  - TXT: 原生支持
- **正则引擎**：原生 RegExp + 自定义规则引擎
