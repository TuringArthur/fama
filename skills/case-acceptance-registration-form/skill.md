# 受理案件登记表

## Overview

受理案件登记表是公安机关工作人员接受立案材料后对刑事案件最初情况的正式记录文书。本技能根据《中华人民共和国刑事诉讼法》第110条第3款的规定，规范生成符合公安机关法律文书标准的受理案件登记表。

## Legal Basis

根据《刑事诉讼法》第110条第3款规定："公安机关、人民检察院或者人民法院对于报案、控告、举报，都应当接受。对于不属于自己管辖的，应当移送主管机关处理，并且通知报案人、控告人、举报人；对于不属于自己管辖而又必须采取紧急措施的，应当先采取紧急措施，然后移送主管机关。"

## Form Structure

### Section 1: 报案人基本信息 (Reporter Information)

| Field | Description | Required |
|-------|-------------|----------|
| 姓名 | 报案人的真实姓名 | Yes |
| 性别 | 男/女 | Yes |
| 年龄 | 报案时的年龄（周岁） | Yes |
| 地址 | 报案人的户籍所在地或常住地址 | Yes |
| 单位 | 报案人所属工作单位或学校 | No |
| 电话 | 报案人联系电话号码 | Yes |
| 报案方式 | 来访/电话/信函/网络/其他 | Yes |

### Section 2: 案件基本情况 (Case Basic Information)

| Field | Description | Required |
|-------|-------------|----------|
| 案件编号 | 公安机关内部案件编号（格式：年份-序列号） | Auto |
| 受理日期 | 案件被受理的具体日期（YYYY-MM-DD） | Yes |
| 发案时间 | 案件发生的具体时间或时间段 | Yes |
| 发案地点 | 案件发生的具体地点，包括省市区街道门牌号 | Yes |
| 案件类型 | 盗窃/抢劫/诈骗/故意伤害/强奸/杀人/毒品犯罪/经济犯罪/其他 | Yes |

### Section 3: 犯罪嫌疑人基本信息 (Suspect Information)

| Field | Description | Required |
|-------|-------------|----------|
| 姓名 | 犯罪嫌疑人的真实姓名 | Conditional |
| 性别 | 男/女 | Conditional |
| 年龄 | 犯罪嫌疑人的年龄 | Conditional |
| 单位 | 犯罪嫌疑人工作单位 | Conditional |
| 住址 | 犯罪嫌疑人户籍所在地或常住地址 | Conditional |
| 特征 | 身高体型、面部特征、衣着打扮等体貌特征 | Conditional |
| 特殊标记 | 纹身、疤痕、胎记等特殊身体标记 | Conditional |

### Section 4: 简要案情 (Brief Case Details)

| Field | Description | Required |
|-------|-------------|----------|
| 受害情况 | 受害人的伤亡情况、人身伤害程度 | Conditional |
| 损失物品 | 被盗、被抢、被骗财物的名称 | Conditional |
| 损失数量 | 损失财物的数量或金额 | Conditional |
| 物品特征 | 损失财物的品牌、型号、颜色、序列号等特征 | Conditional |
| 案发经过 | 案件发生的完整经过描述（不少于200字） | Yes |

### Section 5: 处理结果 (Processing Result)

| Field | Description | Required |
|-------|-------------|----------|
| 初审意见 | 经办人对案件是否应当立案的初步意见 | Yes |
| 处理方式 | 立案侦查/移交管辖/不予立案/调解处理 | Yes |
| 紧急措施 | 是否已采取紧急措施及具体内容 | Conditional |
| 移交说明 | 如移交，移交原因及接收机关 | Conditional |
| 附件清单 | 随案移交的材料清单 | No |

### Section 6: 签名栏 (Signature Section)

| Field | Description | Required |
|-------|-------------|----------|
| 受理人签名 | 受理案件的工作人员签名 | Yes |
| 受理日期 | 签名当天的日期 | Yes |
| 审核人签名 | 部门负责人审核签名 | Conditional |
| 审核日期 | 审核当天的日期 | Conditional |

## Output Format

生成完整的受理案件登记表文档，包含以下格式要素：

1. **文头**：机关名称、文件标题、案件编号
2. **正文**：六大板块内容完整填写
3. **落款**：签名、日期、印章位置标注
4. **附件**：材料清单标注

## Common Edge Cases

### 1. 报案人不愿提供完整信息

处理方式：如实记录报案人自愿提供的信息，在备注栏注明"报案人拒绝提供[具体项目]"，不影响案件受理。

### 2. 犯罪嫌疑人信息不详

处理方式：填写已知信息，其他项目填"不详"，在案发经过中描述目击证言或推断信息。

### 3. 跨区域案件

处理方式：明确记录发案地点，按照管辖规定确定受理机关，如需移交应在处理结果中说明。

### 4. 紧急情况

处理方式：优先采取紧急措施，在备注栏记录措施内容和执行时间，事后补充完整登记表。

### 5. 匿名举报

处理方式：报案人信息填"匿名举报"，其他信息根据举报内容如实填写，注明信息来源渠道。

## Quality Standards

### Content Requirements

- 所有必填字段必须填写完整
- 日期格式统一为YYYY-MM-DD
- 金额使用人民币元为单位
- 地址填写至最小行政区划单位
- 案发经过描述客观真实，避免主观臆测

### Format Requirements

- 文档标题居中，使用加粗宋体
- 表格边框清晰，单元格内容对齐
- 签名处预留足够空间
- 页边距符合公文格式标准
- 总页数控制在2页以内

## Validation Checklist

生成表单后，请确认以下项目：

- [ ] 报案人姓名、电话、报案方式已填写
- [ ] 发案时间和发案地点完整准确
- [ ] 案件类型已选择
- [ ] 案发经过描述清晰完整
- [ ] 受理人已签名并填写日期
- [ ] 初审意见和处理方式已确定
