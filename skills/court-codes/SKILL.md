# 人民法院代字技能

## 概述

本技能提供中国各级人民法院代字的查询功能，支持通过法院名称、地区、行政区划代码等条件快速检索对应的法院代字，并辅助生成符合规范的案件编号（案号）。

## 功能特点

### 1. 法院代字查询
支持查询以下各级法院：
- 最高人民法院
- 高级人民法院（省级）
- 中级人民法院（市级）
- 基层人民法院（区县）
- 专门法院（海事法院、铁路运输法院、知识产权法院、金融法院、互联网法院等）

### 2. 案号格式规范
根据最高人民法院规定，案号格式为：
```
[代字] + [年份] + [案件类型代字] + [序号]
```

**年份**：4位数字，如2026

**案件类型代字**：
| 类型 | 代字 | 说明 |
|------|------|------|
| 刑事案件 | 刑初 | 刑事一审 |
| 刑事案件 | 刑终 | 刑事二审/再审 |
| 民事案件 | 民初 | 民事一审 |
| 民事案件 | 民终 | 民事二审/再审 |
| 行政案件 | 行初 | 行政一审 |
| 行政案件 | 行终 | 行政二审/再审 |
| 执行案件 | 执 | 执行案件 |
| 赔偿案件 | 赔 | 国家赔偿 |
| 再审案件 | 再 | 再审案件 |

**序号**：6位数字，如000001

### 3. 案号生成示例

| 法院 | 年份 | 案件类型 | 生成案号 |
|------|------|----------|----------|
| 北京市第一中级人民法院 | 2026 | 刑事一审 | 京01刑初2026000001号 |
| 上海市浦东新区人民法院 | 2026 | 民事一审 | 沪0105民初2026000001号 |
| 广东省高级人民法院 | 2026 | 刑事二审 | 粤刑终2026000001号 |

## 数据结构

法院代字数据库字段说明：
- **courtName**: 法院全称
- **courtCode**: 法院代字
- **regionCode**: 行政区划代码（6位）
- **regionName**: 对应行政区划名称
- **courtLevel**: 法院级别（1-高级/2-中级/3-基层/4-专门）
- **provinceCode**: 省份代码（2位）
- **cityCode**: 城市代码（4位）

## 使用场景

### 场景1：查询特定法院的代字
用户询问"北京市朝阳区人民法院的代字是什么？"
- 检索关键词：朝阳区人民法院、京
- 返回结果：法院代字为"京0105"

### 场景2：生成案件编号
用户需要生成一份民事判决书的案号
- 确认法院：上海市第二中级人民法院
- 法院代字：沪02
- 生成完整案号：沪02民初2026000001号

### 场景3：确认案号格式
用户不确定案号格式是否正确
- 提供规范格式说明
- 对照用户提供的案号进行校验

## 注意事项

1. **代字唯一性**：法院代字在全国范围内具有唯一性，不可重复
2. **行政区划变更**：部分法院代字可能因行政区划调整而发生变化
3. **专门法院**：专门法院（如铁路、海事）的代字通常在省份代字后加数字
4. **案号编号**：序号每年从000001重新开始编号

## 数据来源

本技能数据来源于《各级法院代字表》附件2，包含：
- 全国各级人民法院基本信息
- 行政区划代码
- 法院级别分类
- 专门法院标识

数据覆盖范围：
- 最高人民法院（1家）
- 高级人民法院（32家，含军事法院等专门法院）
- 中级人民法院（400余家）
- 基层人民法院（3200余家）
- 专门法院（海事、铁路、知识产权、金融、互联网等）
